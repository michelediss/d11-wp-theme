# Cookie Consent

`simple-cookie-consent` is the plugin that handles consent banners, Gutenberg blocks, and cookie registry management in a way that stays aligned with the site's interface system.

## Goal

Cookie consent should not feel like a foreign widget glued on top of the site. It should behave like part of the same compositional logic:

- controlled through blocks
- integrated with the theme
- consistent with the design system
- manageable inside Gutenberg

## What it includes

- a consent banner
- dedicated Gutenberg blocks
- cookie registry management
- support for blocked content until consent is granted

## Why it fits the project

The interface is treated as something that emerges from fundamental combinable structures. The consent layer should therefore behave like part of the system, not like an external overlay with its own unrelated rules.

This plugin makes it possible to:

- keep consent inside the editorial flow of the site
- use reusable interface components
- make compliance, design, and composition work together

## Result

Cookie consent becomes part of the site's interface engine instead of a separate technical exception.
